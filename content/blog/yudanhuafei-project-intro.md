最近把这个小项目重新整理了一遍，也顺手把它的定位想得更清楚了一些。

**鱼蛋花费** 不只是一个“记账工具”，我更希望它是一个轻量的个人生活面板：

- 一边记录每天的收入和支出
- 一边写下自己的想法、复盘和成长

所以现在的它，包含了两个核心模块：**鱼蛋小账本** 和 **成长 Log**。

---

## 为什么会有这个项目？

我一直很喜欢“把生活留下痕迹”这件事。

有些痕迹是数字，比如今天花了多少钱、这个月在哪些地方支出最多；有些痕迹是文字，比如今天学到了什么、最近在想什么、某个阶段的自己正在经历什么。

很多工具只能解决其中一半问题：

- 记账 App 更擅长数字
- 博客系统更擅长内容

但对我来说，这两件事其实都属于 **记录生活**。

于是我想做一个更简单的站点，把它们放在一起：

> 账本负责记录生活的流向，文字负责记录生活的感受。

---

## 两个核心模块

## 📒 鱼蛋小账本

小账本是这个项目里更偏“实用工具”的部分。

它解决的不是复杂财务管理，而是一个更日常的问题：**如何快速、低负担地把生活开销记下来**。

现在支持的能力包括：

- 收入 / 支出记录
- 分类、类型、关键词筛选
- 交易记录查看、编辑、删除
- Telegram Bot 自然语言记账
- 通过 Bearer Token 保护敏感操作

我很喜欢 Telegram Bot 这部分，因为它让“记账”这件事更接近自然动作。

例如：

- `午饭 35`
- `工资 +5000`
- `咖啡 28 饮品`

不需要先打开某个复杂页面，再点一堆按钮，只要像发消息一样，把这一笔支出说出来就行。

这种体验很适合真实生活里的碎片化使用场景。

---

## 🌱 成长 Log

另一个模块是成长 Log。

这是一个很轻的内容系统，没有接数据库，而是直接使用 `content/blog/` 目录下的 Markdown 文件作为内容来源。

我喜欢这种方式，原因很简单：

- 内容就是文件，足够直接
- 不需要额外后台
- 版本可追踪
- 很适合个人写作和长期积累

每篇文章都可以通过 Frontmatter 配置标题、日期、摘要、标签等信息，然后交给页面自动读取和渲染。

现在它已经支持：

- Markdown + Frontmatter
- 按日期倒序展示
- GFM 语法
- 代码高亮
- 静态生成

某种程度上，这个模块更像是一个“数字花园”的起点。

## 这个项目是怎么实现的？

整个项目基于 **Next.js 16 App Router** 构建，主要技术栈包括：

- **TypeScript**：保证代码可维护性
- **Tailwind CSS 4**：快速完成页面样式
- **Radix UI**：补充基础交互组件
- **Supabase PostgreSQL**：存储账本数据
- **react-markdown + gray-matter**：处理博客内容
- **remark-gfm / rehype-highlight / highlight.js**：增强 Markdown 渲染体验
- **Telegram Webhook**：作为自然语言记账的输入入口

从结构上看，它并不是一个很大的系统，但我挺喜欢这种小而完整的组合：

- 有前端页面
- 有 API 路由
- 有数据库
- 有静态内容系统
- 还有一点点 Bot 自动化能力

它的复杂度刚刚好，足够表达想法，也足够继续迭代。

## 如果你想本地跑起来

这个项目本地启动不复杂，大致分成四步：

1. 安装依赖：`npm install`
2. 配置 `.env.local`
3. 在 Supabase 执行 `schema.sql` 初始化 `transactions` 表
4. 运行：`npm run dev`

需要的环境变量有：

- `API_KEY`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `BASE_URL`

如果只想写 blog，其实只要内容模块可用就够了；如果想完整体验账本和 Bot，再把 Supabase 与 Telegram Webhook 配起来即可。

## 我希望它以后长成什么样？

现在的鱼蛋花费还是一个很轻量的版本，但它已经有了一个我很喜欢的雏形。

接下来我想继续做的方向包括：

- 账本统计图表与月度分析
- 更聪明的自然语言解析
- 标签系统与多账本支持
- 博客草稿、置顶、封面图等能力
- 登录系统与多用户隔离

我希望它未来依然保持“小”的感觉：

不是做成一个庞大复杂的平台，而是做成一个真正愿意长期使用、长期记录的个人空间。

## 写在最后

对我来说，**鱼蛋花费** 的意义不只是“能用”，而是它把两件我都很在意的事情放在了一起：

- 对生活的管理
- 对成长的记录

前者让我更清楚钱花去了哪里，后者让我更清楚时间流向了哪里。

如果一个项目能同时留下这两种痕迹，那它就已经很值得被继续做下去了。
